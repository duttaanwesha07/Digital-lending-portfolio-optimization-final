# Workstream H — Sensitivity Analysis

## Why this workstream exists

Every number in Workstream G's scenarios depends on assumptions — what cost of funds we used, what loss given default we assumed, what behavioural cutoff we picked, how aggressive a growth rate we modelled. If any of these assumptions is wrong, the recommendation could collapse, or it could quietly be sub-optimal. Workstream H pressure-tests the recommendation against ranges of each major assumption. It proves which findings are robust and surfaces any that depend on getting an assumption right.

## What this workstream does

Takes the locked recommendation from Workstream G and re-runs the impact computation under multiple flexed assumption values — wider macro stress, higher and lower loss given default, different behavioural cutoffs, different growth rates. Identifies which assumptions actually move the answer (and which barely matter). Looks for break-points where the recommendation flips from net-positive to net-negative. Occasionally catches a sub-optimal choice that the original analysis missed — in which case the finding flows back into Workstream G and the recommendation is updated.

## What it produces

Sensitivity tables for each major assumption, a findings document summarising what is robust and what is fragile, and (if relevant) an identification of the break-point at which the recommendation stops working.

## How it fits with other workstreams

Feeds Workstream I as the defensibility argument — the section of the recommendation that proves to the CRO that the headline number doesn't hinge on guessing one input correctly. Occasionally feeds corrections back into Workstream G.
